# 🔰 Methods — Interview Questions and Answers

---

## 1️⃣ **What is a method in Java?**

**A:** A method in Java is a block of code that performs a specific task and is executed when it is called.

---

## 2️⃣ **Why do we need methods in large-scale applications?**

**A:** Methods help divide large programs into smaller, manageable parts, improving readability, reusability, and maintainability.

---

## 3️⃣ **What happens if we write all logic inside the main method?**

**A:** The code becomes complex, difficult to maintain, and hard to debug, especially in large applications.

---

## 4️⃣ **How do methods improve code reusability?**

**A:** Methods allow the same logic to be reused multiple times without rewriting code.

---

## 5️⃣ **How do methods improve maintainability?**

**A:** Methods make it easier to update and manage code because changes can be made in one place.

---

## 6️⃣ **Can Java programs exist without methods? Why?**

**A:** No, every Java program must have at least one method, which is the main() method, as the entry point.

---

## 7️⃣ **What are the components of a method?**

**A:** Method name, return type, parameters, access modifier, and method body.

---

## 8️⃣ **Is return type part of method signature?**

**A:** No, return type is not part of method signature.

---

## 9️⃣ **What happens if we change parameter order in a method?**

**A:** It is treated as a different method and supports method overloading.

---

## 🔟 **Can a method exist without parameters?**

**A:** Yes, methods can exist without parameters.

---

## 1️⃣1️⃣ **What is method signature?**

**A:** Method signature consists of method name and parameter list.

---

## 1️⃣2️⃣ **Can two methods differ only by return type?**

**A:** No, methods cannot be overloaded based only on return type.

---

## 1️⃣3️⃣ **Why is return type not part of method signature?**

**A:** Because method calls are resolved based on method name and parameters, not return type.

---

## 1️⃣4️⃣ **How does compiler resolve overloaded methods?**

**A:** The compiler uses method name and parameter types at compile time to determine which method to call.

---

## 1️⃣5️⃣ **What is method call stack?**

**A:** It is a stack structure that stores method calls in the order they are executed.

---

## 1️⃣6️⃣ **Why is call stack LIFO?**

**A:** Because the last method called is the first one to complete execution.

---

## 1️⃣7️⃣ **What happens internally when a method is called?**

**A:** A new stack frame is created in memory to store method data, and it is removed after execution completes.

---

## 1️⃣8️⃣ **What is stack overflow?**

**A:** Stack overflow occurs when the call stack exceeds its memory limit due to too many method calls.

---

## 1️⃣9️⃣ **Give a real-world example of stack overflow**

**A:** Infinite recursion without a base condition can cause stack overflow.

---

## 2️⃣0️⃣ **How does recursion use call stack?**

**A:** Each recursive call creates a new stack frame until the base condition is met.

---

## 2️⃣1️⃣ **What are types of methods in Java?**

**A:** Static methods, instance methods, abstract methods, and final methods.

---

## 2️⃣2️⃣ **What is difference between predefined and user-defined methods?**

**A:** Predefined methods are built-in (like Math.sqrt()), while user-defined methods are created by programmers.

---

## 2️⃣3️⃣ **What is an abstract method?**

**A:** An abstract method is a method declared without implementation and must be implemented by subclasses.

---

## 2️⃣4️⃣ **Give real-world examples of different method types**

**A:** Static → utility functions, Instance → object behavior, Abstract → design templates.

---
